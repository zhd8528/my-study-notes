# 运维知识库

## 1. 员工电脑上不了网，你的排查思路是什么？

**采用分层排查思路，从本机 -> 局域网 -> 外网 逐级定位**：

1. 打开 cmd，`ipconfig`查看本机 IP，子网掩码、网关地址
2. `ping 127.0.0.1`，不通代表本机网卡驱动异常1
3. ping 网关地址，不通就是局域网线路/路由器故障
4. ping 百度域名，能 ping 同 IP、不通域名就是 DNS 配置问题

## 2. 员工打印机无法打印，常规排查步骤？

1. 先确定打印机与电脑在同一局域网，检查打印机开机、网络/WIFI 连接
2. 核对电脑端驱动是否匹配，异常就重装打印机驱动，重新添加网络打印机
3. 测试打印测试页，仍不行查看打印机 IP、共享权限设置

## 3. 电脑开机蓝屏怎么处理？

1. 询问近期是否新装软件、驱动、卸载可疑程序
2. 进安全模式检查磁盘坏道、更新/回滚显卡驱动
3. 以上无效再备份数据重装系统

## 4. 如果多个员工同时电脑故障忙不过来怎么安排？

1. 管理层、业务在岗办公故障优先处理
2. 普通员工问题登记台账、按排队顺序延后处理
3. 简单问题发图文教程引导员工自行操作，提高效率

## 5. 公司需要批量给几十台电脑装办公软件，你怎么做？

1. 准备统一安装包，借助远程工具批量静默推送安装
2. 机器较多可做系统镜像，批量部署提升效率，减少重复操作

## 6. 员工电脑提示 IP 地址冲突怎么解决？

1. 先查看局域网 DHCP 服务状态，若自动获取冲突：优先由路由/交换机规范 DHCP 地址池
2. 少量终端临时解决：改成静态 IP，避开已占用网段
3. 事后登记，避免后续重复冲突

## 7. 员工电脑 C 盘爆满卡顿，怎么处理？

1. 先用系统磁盘清理、清理临时文件、回收站、下载文件夹释放空间
2. 转移桌面、文档、安装包等个人文件至其他盘符
3. 卸载不常用软件，必要时扩容分区

## 8. 同事需要新装软件，但没有管理员权限怎么办？

1. 我用管理员账号远程协助安装所需软件
2. 登记软件安装记录，原则上不向普通员工下放管理员权限，规避系统安全风险

## 9. 办公网络突然大面积上不了网，排查思路？

1. 先排查上联光猫、主路由指示灯，重启主路由测试
2. 单片区断网查对应交换机，全公司断网优先确认外网进线与运营商链路
3. 查看是否 IP 网段、DHCP 异常或带宽拥堵

## 10. 有人私自改 IP 导致局域网异常，怎么处理？

1. 先把违规 IP 改回 DHCP 自动获取或规范静态 IP，恢复网络
2. 告知员工私自修改 IP 的影响与规章制度
3. 长期可在交换机绑定 IP + MAC，杜绝随意改 IP

## 11. 服务器磁盘使用率过高告警，怎么处理？

1. 先用`df -h`、`du -sh *`定位占用高的目录与大文件
2. 清理无用日志、过期备份、垃圾文件释放空间
3. 确认无用服务再关停，必要时扩容磁盘，避免随意关停业务服务

## 12. 远程连接不上分公司电脑，排查顺序？

1. 先确认对方电脑开机、内网连通，ping 测连通性
2. 检查目标机远程服务是否开启、防火墙放行对应端口
3. 核对账号权限，权限不足则申请开通

## 13. DHCP 分配地址不全、部分设备拿不到 IP 如何排查？

1. 先查看 DHCP 地址池是否耗尽，地址用完就扩容网段
2. 检查服务器/路由 DHCP 服务状态，异常再重启服务
3. 排查网关、交换机是否拦截 DHCP 报文，排除环路干扰

## 14. 同一个办公室一半能上网一半不能，排查顺序？

1. 定位故障网段对应的接入交换机，查看规则设备所在端口组指示灯
2. 更换故障端口/网线，排查 VLAN 配置、端口是否被禁用
3. 检查该交换机上联口故障或 DHCP 分区异常

## 15. 服务器 ping 网关通，但无法访问外网，排查？

1. 检查默认路由配置是否缺失/错误，网关路由异常无法转发外网
2. 核查 NAT 策略、防火墙是否拦截外网出网流量
3. 核对 DNS 配置，DNS 失效也会出现能 ping 网关但打不开网页

## 16. 外网带宽卡顿、网速忽快忽慢排查？

1. 登录网关/防火墙查看带宽监控，排查是否个别终端跑满带宽、异常下载
2. 核对端口限速、QoS 策略配置，确认有无异常限流
3. 排查内网环路、ARP 攻击占用带宽，必要时做流量分流管控

## 17. Linux 服务器忘记登录密码怎么重置？

1. 服务器重启进入 grub 引导菜单，编辑内核启动参数进入单用户/救援模式，不是 BIOS
2. 挂载根分区后修改 `/etc/shadow`重置 root 密码
3. 改完重新挂载、正常启动验证登录

## 18. 内网某主机持续大量发包造成内网风暴，处理步骤？

1. 交换机上找到故障主机端口，立即 shutdown 端口隔离断网，终止风暴
2. 通过 MAC 地址定位设备，排查病毒、网卡故障或环路
3. 修复后再接回网络，可做端口风暴抑制配置规避复发

## 19. Linux 查看端口占用进程？

1. `netstat -tulnp | grep 端口`
2. `ss -tulnp | grep 端口`
3. 查到 PID 后再用 `ps aux | grep PID`看进程详情

## 20. 服务器 CPU 满载居高不下排查？

1. 先用 `top/htop`按 CPU 排序，定位占用过高 PID 与进程名
2. 区分业务进程和异常进程，业务进程不能随意关停
3. 异常无用进程 kill 处理，若程序异常卡死则排查代码/日志

## 21. Linux 磁盘只读无法写入怎么排查？

1. 先查看日志：`dmesg`，大多是磁盘报错触发系统自动挂载只读保护数据
2. 不是单纯改权限，尝试重新挂载：`mount -o remount,rw /`
3. 挂载失败则检查磁盘坏道、文件系统，执行`fsck`修复

## 22. 配置 Linux 开机自启动定义脚本？

1. systemd 服务：编写 service 文件，`Restart=always`配置进程异常自动重启，放入`/etc/systemd/system/`，`systemctl enable xxx`开机自启
2. contrab@reboot：`@reboot /xxx/脚本.sh`，开机直接执行

## 23. 防火墙放行单个端口外网访问？

1. centos/firewalld：`firewall-cmd --add-port=端口/tcp --permanent && firewall-cmd --reload`
2. iptables：`iptables -A INPUT -p tcp --dport 端口 -j ACCEPT && 保存规则`
3. 云服务器还要在厂商控制台安全组放行对应端口

## 24. 查看 Linux 系统开机启动耗时？

1. `systemd-analyze`：查看整体开机总耗时
2. `systemd-analyze blame`：按耗时排序各开机服务
3. `journalctl -b`：查看本次开机完整系统日志

## 25. Linux 查找服务器中包含指定内容的文件？

1. 文件名查找（通配符）：`find/目录 -name "*.txt"`
2. 文件内容搜关键字：`grep -r "关键词" /目录`

## 26. 压缩解压 tar.gz 包命令

- 压缩：`tar -zcvf 打包名.tar.gz 源文件/目录`（c创建）
- 解压：`tar -zxvf xxx.tar.gz`（x创建）

## 27. 查看 Linux 磁盘使用率？

- `df -h`：查看分区磁盘占用
- `du -sh *`：查看当前目录各文件夹大小

## 28. 查看内存使用？

- 查看内存：`free -h`、`top`、`htop`

## 29. 创建 Linux 普通用户并设置密码？

```bash
useradd testuser
passwd testuser
```

- useradd 创建用户，passwd 配置登录密码

## 30. 删除 Linux 用户连同家目录？

```bash
userdel -r 用户名
```

- `-r`：同步删除用户家目录与邮件文件
- `rm -rf`只手动删除文件夹，没删系统用户信息

## 31. Linux 切换用户？

1. `su - 用户名`：输入目标用户密码切换
2. `sudo -i`：切 root（需要当前用户 sudo 权限）
