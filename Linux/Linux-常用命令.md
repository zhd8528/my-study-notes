# Linux-常用命令

## ls（list）显示目录内容

```b
# Linux 中的命令一般都是由命令名和它后面的一些参数组成的
# 基本上所有的命令都可以接受一个或者多个参数
# 参数的作用是用来指定命令的具体操作
ls -l
# -l 可以显示更详细的信息，包括文件的权限、所有者、大小、修改时间等
ls -a
# -a 可以显示包括隐藏文件在内的所有文件和目录，在 Linux系统中，以半角英文的点`.`开头的文件和目录都是隐藏的
ls -h
# -h 可以以人类可读的方式显示文件的大小
ls -t
# -t 可以按照修改时间排序
ls -r
# -r 可以逆序显示
……
# 这些参数也可以组合在一起使用
```

## ln（link）创建链接文件

```b
# 左边这一列是文件的类型和权限：
# 第一个字符表示文件的类型，中杠（-）表示普通文件，d 表示目录，l 表示链接文件
# Linux 系统的一种特殊文件，分为硬链接和软链接两种
# 软链接也叫符号链接，类似于 Windows 系统中的快捷方式： 链接文件本身 -> 指向的文件或者目录
# 创建链接文件的命令是 ln（link），后面加上源文件的名称以及链接文件的名称
ln -s
# -s 表示创建的是一个软链接文件
# 软链接本身其实并不存储文件的内容，只是一个指向目标文件的快捷方式，所以大小非常小
# 不加参数默认是硬链接，硬链接本质上是一个指向文件系统中另一个文件的指针，它与原始文件共享相同的 i 节点
```

## inode（index node）

- 文件类型：例如，是一个普通文件、目录还是链接
- 权限：文件的访问权限，如读、写和执行权限
- 所有者：文件的拥有者和组信息
- 文件大小：文件的大小（字节数）
- 时间戳：文件的创建时间、最后访问时间和最后修改时间
- 数据块指针：指向实际存储文件数据的磁盘块的指针

```b
# i 节点是文件系统中一个非常重要的概念
# 在 Linux 系统中，每个文件或者目录都有一个唯一的 i 节点，它存储了文件的元数据信息，包括文件的权限、所有者、大小、修改时间等
# 系统也是通过这个 i 节点来识别文件的
# 硬链接和原始文件共享相同的 i 节点，也意味着它们共享相同的文件内容
# 源文件和硬链接实际上是文件系统中，同一个文件的两个不同的名字
```

```b
echo "Hello Linux" > file2.txt
# 使用 echo 命令创建一个文件，> 表示把 echo 命令的输出重定向到一个文件中，后面加上文件的名称
ls -ltri
# -i 查看文件的 i 节点，硬链接的 i 节点（最前面的数字）是一样的
# 当用 echo 修改一个文件的内容，另一个文件的内容也随着发生变化，源文件和链接文件的大小和修改时间是一样的
# 列表中第 2 列的数字表示文件的硬链接数
# 当删除其中一个文件的时候，另一个文件还是可以正常访问的
rm file2.txt
# 使用 rm 命令删除文件
# 当把软链接的目标文件删除的时候，软链接文件就会失效，链接文件变成红色表示正指向一个并不存在的文件
#还有另外一个区别，软链接可以指向文件或者目录，硬链接只能指向文件，不能指向目录
```

## 文件权限

```b
# 在 ls 命令回显的列表中，最左边的 9 个字符表示文件的权限
# 可以按照 3 个字符一组来看
# 第一组表示的是文件的所有者（user）的权限
# 第二组表示的是文件所属组别（group）的权限
# 第三组表示的是其他用户（other）的权限
# 每一组中的 3 个字符分别表示的是读（r 可读）、写（w 可写）和执行（x 可执行）权限
# 而相应的位置上如果没有相应的权限就会显示一个中杆（-）
```

## chmod（change mode）修改权限

```b
# 可以使用chmod 命令来修改文件的权限
chmod +x hello1.txt
# 加上 +x 表示添加可执行权限，文件的权限会发生变化，文件变成绿色表示是一个可执行文件
# 想要删除权限是是 -x
# 还可以在加号或减号前面加上u、g或o来单独给文件的所有者、同组或者其他用户来添加或删除权限，也可以组合在一起使用
```

## touch 命令创建文件

```b
# 在 Linux 中，创建文件有很多中方式，比如用 vi 编辑器
touch hello.txt
# 可以使用 touch 命令创建文件，这个命令本来是用来更新文件的修改时间的，但是如果文件不存在就会创建一个新的空文件
echo "文件内容" > hello.txt
# echo 是一个用来输出文本的命令,可以使用重定向符号 > 把文本内容保存的文件中，当文件不存在会自动创建一个新的文件
```

## pwd（print working directory）& cd（change directory）

```b
# 在 Windows 系统中，文件和目录是以盘符开始的，比如 C 盘、D 盘
# 在 Linux 系统中，文件系统上一个树状结构，所有的文件和目录都是从根目录开始的
# 根目录的符号是一个斜线（/）
pwd
# 我们可以执行 pwd 命令，它的作用是显示当前所在目录的位置
cd /
# 我们也可以使用 cd 命令来切换目录
# Linux 中的路径有两种类型，一种是绝对路径，一种是相对路径
# 绝对路径是从根目录开始的完整路径，比如 /root
# 相对路径就是指相对于当前目录的路径，可以使用一个点 . 表示当前目录，两个点 .. 来表示上一层目录
cd ~
# ~ 表示当前用户的家目录
# - 表示上一次所在的目录
```

## 根目录下一些常见目录以及它们的作用

```b
# bin（binary） 目录存放的是系统的基本命令（比如ls，cat），以及一些二进制的可执行文件
# etc 目录存放的是系统和一些软件的配置文件
# home 目录一般是用户的家目录，当进入系统之后默认的目录
```

![image-20260612205936364](C:\Users\Administrator\Desktop\my-study-notes\Linux\image-20260612205936364.png)

- /：根目录，所有文件和目录的起点
- /bin：用户二进制文件
- /sbin：系统二进制文件
- /etc：配置文件
- /dev：设备文件
- /home：用户家目录
- /root：root 用户家目录
- /lib：系统库文件
- /opt：可选的第三方软件包
- /tmp：临时文件
- /usr：用户程序
- /var：可变文件如日志等
- /boot：启动加载器文件
- /proc：进程信息
- /sys：系统文件
