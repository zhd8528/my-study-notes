## 追问

### 1. TCP 四次挥手细节

**为什么需要四次挥手？**

> TCP 是全双工的，双方都要关闭各自的连接
>
> 主动方发送 FIN 表示不再发送数据，但还能接收；
>
> 被动方先回复 ACK，等自己发完数据后再发 FIN

**TIME_WAIT 为什么是 2MSL？**

> MSL 是报文最大生存时间
>
> 2MSL 确保被动方收到最后的 ACK，并且让旧连接的所有报文都从网络中消失，防止影响旧连接

**CLOSE_WAIT 和 TIME_WAIT 哪个是主动方的状态？**

> TIME_WAIT 是主动发送方（先发 FIN）；
>
> CLOSE_WAIT 是被动关闭方（收到 FIN 后回复 ACK，等自己发 FIN）

**大量 TIME_WAIT 有什么问题？怎么解决？**

> 占用端口资源，可能导致无端口可用
>
> 解决方案：
>
> 调小 net.ipv4.tcp_fin_timeout、开启 tcp_tw_reuse、调整端口范围

### 2. TCP 与 UDP 的区别

| 对比项 | TCP              | UDP                 |
| ------ | ---------------- | ------------------- |
| 连接   | 面向连接         | 无连接              |
| 可靠性 | 可靠（确认重传） | 不可靠（可能丢包）  |
| 速度   | 慢               | 快                  |
| 应用   | HTTP、FTP、SSH   | DNS、视频通话、游戏 |

**为什么视频用 UDP？**

> 实时性要求高于可靠性，丢一帧无所谓，但延迟不能高

---

### 3. TCP 如何保证可靠传输

**机制**：

> **1. 序列号 + 确认应答**：
>
> 每个字节有编号，ACK 告知已收到
>
> **2. 超时重传**：
>
> 发送后若超时未收到 ACK，则重传
>
> **3. 流量控制（滑动窗口）**：
>
> 接收方告诉发送方自己的缓冲区大小，防止发太快
>
> **4. 拥塞控制**：
>
> 慢启动、拥塞避免、快重传、快恢复

### 4. HTTP 状态码补充

| 状态码                 | 含义           | 场景                                  |
| ---------------------- | -------------- | ------------------------------------- |
| 201 Created            | 创建成功       | POST 提交数据后返回                   |
| 204 No Connect         | 成功但无返回体 | PUT 修改后常用                        |
| 304 Not Modified       | 缓存未修改     | 条件请求，客户端可直接用缓存          |
| 307 Temporary Redirect | 临时重定向     | 与 302 类似，但不允许改变 POST 为 GET |
| 429 Too Many Requests  | 请求过多       | 限流返回                              |

### 5. GET VS POST 补充

**什么是幂等？**

> 一次或多次请求，资源状态相同
>
> GET是安全的也是幂等的；
>
> POST 不安全也不幂等（多次提交会创建多个资源）

**PUT 和 DELETE 的幂等性？**

> PUT 幂等（更新相同数据多次结果一样）；
>
> DELETE 幂等（删除多次结果一样）

### 6. HTTPS 握手过程

**HTTPS 如何保证安全？**

> 混合加密：
>
> 非对称加密传输/对称密钥，对称加密传输数据

**简要握手过程？**

1. 客户端发支持的加密套件
2. 服务端发证书（公钥）
3. 客户端验证证书，生成随机密钥（用公钥加密）发送
4. 服务端解密得到对称密钥
5. 后续通信用对称加密

**证书的作用？**

> 防止中间人攻击，证明服务端身份（CA 签发）

---

### 7. HTTP 版本演进

**HTTP/1.0**：

> 短连接，每次请求重新建立 TCP

**HTTP/1.1**：

> 持久连接（keep-alive）、管道化（但仍有队头阻塞）

**HTTP/2**：

> 二进制分帧、多路复用、服务器推送、头部压缩

**HTTP/3**：

> 基于 UDP 的 QUIC 协议，解决队头阻塞，连接迁移

### 8. Cookie 与 Session

**Cookie 和 Session 区别？**

> Cookie 存客户端，Session 存 服务端
>
> Session 通过 Cookie 中 的 sessionId 关联

**禁用 Cookie 怎么办？**

> URL 重写（把 sessionId 拼在 URL 里）
