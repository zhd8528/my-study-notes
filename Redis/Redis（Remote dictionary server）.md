## 简介

> Redis 是一个开源的基于内存的数据存储系统，它可以用作数据库（DB）、缓存（Cache）和消息队列（MQ）等各种场景
>
> 它也是目前最热门的 NoSQL 数据库之一
>
> 早期互联网公司的应用系统大多是通过 MySQL 这种传统的数据库来对外提供服务
>
> 随着互联网的快速发展，应用系统的访问量越来越大，数据库的性能瓶颈越来越明显，主要是磁盘 IO 所导致的
>
> 磁盘 IO 的读写操作速度与内存相比是非常慢的
>
> 如果能够把数据存储在内存中的话，就可以大大提高它的性能了
>
> 于是就有了 Redis 这种基于内存的数据存储系统
>
> Redis 支持多种数据结构，包括 5 种基本数据类型 和 5 种高级数据类型

**5 种基本数据类型**：

- 字符串 String
- 列表 List
- 集合 Set
- 有序集合 SortedSet
- 哈希 Hash

**5 种高级数据类型**：

- 消息队列 Stream
- 地理空间 Geospatial
- HyperLogLog
- 位图 Bitmap
- 位域 Bitfield

> Redis 的使用方式主要包括 3 种类型

- 命令行界面：CLI（Command Line Interface）
  - 通过 Redis-CLI 命令行工具来使用 Redis
- 应用程序接口：API（Application Programming Interface）
  - 使用 Java 或者 Python 等常用的编程语言，通过编写代码的方式来使用 Redis
- 图形用户界面：GUI（Graphical User Interface）
  - 通过一些图形化工具来使用 Redis，比如 RedisInsight

> 最主要的是 Redis 的性能非常地高
>
> 它和其他数据库相比，性能差了一个数量级
>
> 所以在互联网领域，Redis 是非常受欢迎的

**Redis 的优势**：

- 性能极高
- 数据类型丰富，单键值对最大支持 512M 大小的数据
- 简单易用，支持所有主流编程语言
- 支持数据持久化、主从复制、哨兵模式等高可用特性

## 字符串 String

> Redis 中的数据是以键值对的形式存储的，所以我们需要指定一个键和一个值
>
> 键和值之间使用空格来分隔
>
> 可以使用 SET 命令来设置一个键值对
>
> 注意 Redis 中的键是区分大小写的
>
> Redis 中默认都是使用字符串来存储数据的，而且是二进制安全的
>
> 这就意味着可以把很多类型的数据都存储在 Redis 中，比如数字、布尔值，甚至是序列号之后的对象等
>
> 可以使用 DEL 命令来删除一个键，用 GET 命令获取被删除的键的值是 (nil)，表示这个键已经被删除掉了
>
> 可以使用 EXISTS 命令判断一个键是否存在（0 不存在，1 存在）
>
> 使用 KEYS 命令查看数据库中有哪些键，后面加上一个 pattern 模式匹配的字符串参数（* 表示查找所有的键，*me 表示查找所有以 me 结尾的键）

![image-20260613155501443](image-20260613155501443.png)

> 可以使用 FLUSHALL 命令把所有数据库里面的键都删除掉（慎用）
>
> Redis 中的键和值都是以二进制的形式存储的，所以默认不支持中文，会把中文换成二进制的形式（\x 表示后面的内容是十六进制的）
>
> 使用 quit 命令退出客户端，登录时加上 --raw 参数，表示以原始的形式来显示内容，就可以显示中文了
>
> 跟 Linux 一样可以用 clear 命令·清空·屏幕

![image-20260613160147843](image-20260613160147843.png)

> 可以用 TTL（Time To Live，表示这个键还有多久过期）命令来查看一个键的过期时间，-1 表示没有过期时间
>
> 可以使用 EXPIRE 命令设置一个键值对的过期时间，-2 表示已经过期了
>
> 也可以使用 SETEX 命令来设置一个带有过期时间的键值对
>
> 还可以使用 SETNX 命令，只有当键不存在时才设置键的值，如果键已经存在，那么 不做任何动作

![image-20260613161043725](image-20260613161043725.png)

## 列表 List

> List 一般用来存储和操作一组有顺序的数据，和数组的概念比较类似
>
> 可以使用 LPUSH 或者 RPUSH 命令将元素添加到列表的头部或者尾部
>
> 如果想要获取列表内容的话，要使用 LRANGE命令：
>
> `LRANGE 列表名称 起始位置 结束位置`，起始位置和结束位置都是以 0 开始，-1 表示最后一个元素
>
> `LRANGE 列表名称 0 -1`就可以获取从第一个到最后一个，也就是列表中所有的元素
>
> LPUSH 也可以一次性添加多个元素，注意在列表中顺序是反过来的
>
> 因为 LPUSH 命令会把参数中的元素按照从左到右的顺序依次添加到列表的头部
>
> 所以最后添加的元素就会排在最前面
>
> 也可以使用 RPUSH 命令从列表的尾部添加元素

![image-20260613164015817](image-20260613164015817.png)

> 可以使用 LPOP 和 RPOP 命令来分别从列表的头部和尾部删除元素
>
> LPOP 和 RPOP 也可以一次性删除多个元素，后面加上数字表示要删除的元素个数：
>
> `LPOP/RPOP 列表名称 要删除的元素个数`
>
> 也可以使用 LLEN 命令查看列表的长度
>
> 这几个命令就可以实现一个简单的消息队列了：
>
> 使用 LPUSH 在列表头的方向添加一个元素，然后使用 RPOP 在列表尾部删除一个元素
>
> 这样就实现了一个最简单的先进先出队列
>
> 可以使用 LTRIM （trim 修剪）命令来删除列表中指定范围以外的元素，只保留 start 和 stop 之间的元素，不在这个范围内的元素都会被删除

![image-20260613164052136](image-20260613164052136.png)

##  集合 Set

> SET 是一种无序集合
>
> 它和列表的区别在于：列表中的元素都是可以重复的，而 SET 中的元素是不可以重复的
>
> 而且 SET 中的元素并不像列表那样是具有顺序的
>
> SET 相关的命令都是以 S 开头的
>
> 比如可以使用 SADD 命令来向 SET 中添加一个元素，后面加上一个键表示集合的名称，然后是要添加到这个机会里面的元素，可以有一个，也可以有多个（1 添加成功，0 添加失败）
>
> 使用 SMEMBERS 命令查看集合中的元素
>
> 使用 SISMENBER 命令判断一个元素是否在集合中（1 在，0 不在）
>
> 使用 SREM 命令删除集合中的元素

![image-20260613190404050](image-20260613190404050.png)

> Redis 支持集合的运算：交集、并集等
>
> SINTER、SUNION、SDIFF 等命令

## 有序集合 SortedSet

> SortedSet 也叫 ZSet，它和集合的区别是每个元素都会关联一个浮点类型的分数，然后按照这个分数来对集合中的元素进行**从小到大**的排序
>
> 有序集合的成员是唯一的，但是分数是可以重复的
>
> 有序集合的相关命令都是以 Z 开头的
>
> 可以使用 ZADD 命令来向有序集合中添加一个元素，后面加上键名和元素
>
> 每个元素都是由一个分数和一个成员组成，分数在前，成员在后，中间用空格隔开
>
> 使用 ZRANGE 命令查看这个有序集合中的元素，后面加上键名，然后是起始位置和结束位置，只输出成员
>
> 如果想要输出分数可以在命令后面加上 WITHSCORES
>
> 如果想要查看某个成员的分数的话，可以使用 ZSCORE 命令
>
> 如果想要查看某个成员的排名的话，可以使用 ZRANK 命令
>
> 排名默认从小到大，但是实际上一般的排名都是从大到小的，可以使用 ZREVRANK 命令

![image-20260613212239683](image-20260613212239683.png)

> 删除集合中的某个成员使用 ZREM 命令，使用 ZREMRANGEBYRANK 命令某个排名范围之内的成员
>
> 使用 ZINCRBY 命令对某个成员的分数进行增加
>
> 使用 ZRANGEBYSCORE 命令 查找某个分数范围之内的成员

![image-20260613213312817](image-20260613213312817.png)

## 哈希 Hash

> 哈希是一个字符类型的字段和值的映射表，简单来说就是一个键值对的集合，特别适合用来存储对象
>
> 哈希相关的命令都是以 H 开头
>
> 可以使用 HSET 命令来向哈希中添加一个键值对，后面加上键名，然后是要添加的键值对
>
> 可以使用 HGET 命令来获取哈希中的某个键值对
>
> 可以使用 HGETALL 获取整个哈希中的所有键值对
>
> 可以使用 HDEL 命令删除哈希中的某个键值对
>
> 可以使用 HEXISTS 命令判断某个键值对是否存在（0 不存在，1 存在）
>
> 可以使用 HKEYS 命令获取哈希中的所有键，HVALS 获取所有值
>
> 可以使用 HLEN 命令获取哈希中所有键值对的数量

## 发布订阅模式

> Redis 提供了发布订阅的功能
>
> 可以通过 PUBLISH 命令将消息发送到指定的频道，后面加上频道的名称和消息的内容
>
> 可以通过 SUBSCRIBE 命令订阅这个频道，就可以接收到这个频道的消息了，订阅的频道终端可以有多个
>
> 但是发布订阅功能还有一些局限性，比如消息无法持久化、无法记录历史信息等，可以用 Stream 流来解决

## 消息队列 Stream

> Stream 是 Redis 5.0版本中引入的一个新的数据结构，它是一个轻量级的消息队列
>
> 可以解决消息无法持久化、无法记录历史消息等问题
>
> Stream 相关的命令都是以 X 开头
>
> 可以通过 XADD 命令向 Stream 中添加消息，后面加上键名，* 表示自动生成一个消息的 ID（Redis 保证是生成的  ID 是自增的），然后是添加消息的名字和内容（键值对）
>
> 添加后的回显信息就是消息的 ID（可以手动加，ID 的格式是一个整数加上短横线，再加上一个整数；第一个整数表示时间戳，第二个整数表示一个序列号，需要自己保证 ID 的递增）
>
> 可以通过 XLEN 命令查看 Stream 中消息的数量
>
> 可以通过 XRANGE 命令查看 Sream 中的消息的详细内容，开始和结束可以使用 - 和 +
>
> 可以使用 XDEL 命令来删除消息，后面加上键名，然后是消息 ID
>
> 可以通过 XTRIM 命令删除消息，后面加上 MAXLEN 0 表示删除所有的消息

![image-20260613221339478](image-20260613221339478.png)

> 消息创建之后需要消费
>
> 可以使用 XREAD 命令读取消息，后面加上 COUNT，比如 COUNT 2 表示一次读取两条消息；然后加上 BLOCK 1000 表示没有消息的话就阻塞 1000 ms（1 秒）；然后加上 STREAMS，后面加上消息队列的名称，0 表示从头开始读取（如果是比消息队列中最大的 ID 还大的一个值，会阻塞 1 秒返回 nil），$ 只希望获取从现在开始以后的最新消息）
>
> 消息可以重复读取

![image-20260613223346290](image-20260613223346290.png)

> 还可以使用 XGROUP 命令创建一个消费者组，在消费者组中添加消费者，后面加上 CREATE，然后加上消息的名称，后面是组的名称，然后是一个 ID
>
> 可以通过 XINFO GROUPS 命令查看消费者组的信息，会返回组的名称、消费者的数量、待处理的消息数等
>
> 可以使用 XGROUP CREATECONSUMER 命令添加消费者，然后加上消息的名字、组的名字，之后是消费者的名字
>
> 可以通过 XREADGROUP 命令读取消息，后面是组的名称、消费者的名称，然后加上消息的数量，COUNT 2 表示一次读取两条消息，后面是 BLOCK 加阻塞的时间，最后是消息的名字（频道的名字），右箭头（>）表示从这个消息中读取最新的消息，回车后两条消息就被消费了
