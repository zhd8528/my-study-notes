## 1. 进程与线程的区别

| 对比项   | 进程                                                | 线程                                           |
| -------- | --------------------------------------------------- | ---------------------------------------------- |
| 资源     | 拥有独立的内存空间、文件描述符等资源                | 共享进程的内存空间和资源                       |
| 创建开销 | 大（需要分配独立地址空间）                          | 小（只需创建栈、寄存器等）                     |
| 通信方式 | 进程间通信（IPC）：管道、消息队列、共享内存、Socket | 直接读写共享内存（注意同步）                   |
| 调度     | 操作系统调度的基本单位                              | CPU 调度的基本单位（同一进程内多个线程可并发） |
| 独立性   | 进程间相互隔离，一个崩溃不影响其他                  | 一个线程崩溃可能导致整个进程崩溃               |
| 系统开销 | 上下文切换开销大                                    | 上下文切换开销小                               |

**总结**：

> 进程是资源分配的最小单位，线程是 CPU 调度的最小单位

## 2. 死锁的四个必要条件

**互斥**：

> 资源一次只能被一个线程/进程占用

**持有并等待**：

> 一个进程至少持有一个资源，且在等待获取其他资源时不释放已有资源

**不可剥夺**：

> 资源只能由持有者主动释放，不能被强制夺走

**循环等待**：

> 进程间形成等待环路：
>
> P1 等 P2 的资源，P2 等 P3 的资源，……，Pn 等 P1 的资源

**追问：如何预防死锁？**

> 破坏四个条件之一
>
> 例如：
>
> 破坏“持有并等待”：
>
> 一次性申请所有资源；
>
> 破坏“不可剥夺”：
>
> 允许剥夺；
>
> 破坏“循环等待”的典型方案：
>
> 所有进程按相同顺序申请资源（如所有进程先申请资源 A 再申请 B）

**死锁的处理策略**（避免 VS 预防 VS 检测）

| 策略       | 方法                               | 代表                     |
| ---------- | ---------------------------------- | ------------------------ |
| 预防       | 破坏四个必要条件之一               | 按顺序申请资源           |
| 避免       | 运行时动态判断，避免进入不安全状态 | 银行家算法               |
| 检测与恢复 | 允许死锁发生，检测并恢复           | 进程回滚、杀死进程       |
| 忽略       | 鸵鸟算法（很多 OS 采用）           | 因为死锁罕见，处理代价高 |

## 3. 虚拟内存与分页

**什么是虚拟内存？**

> 每个进程看到连续的、独立的地址空间（虚拟地址），由操作系统映射到物理内存
>
> 隔离进程，支持大程序运行

**分页是什么？**

> 将虚拟地址空间和物理内存都分成固定大小的页（如 4KB）
>
> 通过页表映射，允许物理内存不连续
>
> 实现按需加载

**分页 VS 分段**：

| 对比项   | 分页                 | 分段                     |
| -------- | -------------------- | ------------------------ |
| 划分方式 | 固定大小（4 KB）     | 可变大小（按逻辑段）     |
| 用户视角 | 不可见               | 可见（代码段、数据段等） |
| 地址空间 | 一维（单一地址空间） | 二维（段号 + 段内偏移）  |
| 外部碎片 | 无                   | 有                       |
| 内存共享 | 难                   | 易（按段共享）           |

**段页式结合**：

> 先分段，段内再分页，兼顾分段共享和分页节省空间

**缺页中断**：

> 访问的页不在物理内存中，触发中断，操作系统从磁盘加载该页，可能换出其他页

**缺页中断处理流程**：

1. CPU 访问虚拟地址，MMU 查询页表
2. 发现页表项无效（不在物理内存）
3. 触发缺页中断，陷入内核态
4. 操作系统检查地址是否合法
   - 非法 -> 段错误（Segmentation Fault）
   - 合法 -> 从磁盘读取页面
5. 如果物理内存已满，执行页面置换算法（LRU、Clock 等）
6. 从磁盘加载页面到物理内存
7. 更新页表项
8. 返回用户态，重新执行触发缺页的指令

**常见页面置换算法**：

| 算法         | 原理                 | 优缺点                              |
| ------------ | -------------------- | ----------------------------------- |
| FIFO         | 先进先出             | 简单但可能置换常用页（Belady 异常） |
| LRU          | 最近最少使用         | 效果好但实现成本高                  |
| Clock（NRU） | 循环扫描，检查访问位 | 近似 LRU，性能可接受                |
| LFU          | 最不经常使用         | 可能让历史热点页长期驻留            |

**Belady 异常**：

> FIFO 算法下，增加物理内存反而导致缺页增多

## 4. 用户态 VS 内核态

**为什么分用户态和内核态？**

> 保护系统安全
>
> 内核态可以执行特权指令（如 I/O，中断），用户态受限
>
> 系统调用切换

**什么时候切换？**

> 系统调用
>
> 异常（如缺页）
>
> 中断

**用户态 <-> 内核态切换成本**：

- 保存/恢复上下文（寄存器、栈等）
- CPU 缓存可能失效
- 切换时间约 100 ns ~ 1μs （比函数调用贵很多）

**减少切换的方法**：

- 批量系统调用（如 readv/writev）
- 用户态库（如VDSO）
- 异步 IO

## 5. 进程的状态

**进程 5 状态模型**：

![进程状态](https://zhd-blog.oss-cn-beijing.aliyuncs.com/3308e10c1180a81ae5504e93f93f4ce0.png)

| 状态 | 说明                   |
| ---- | ---------------------- |
| 创建 | PCB 已分配，资源未分配 |
| 就绪 | 等待 CPU 调度          |
| 运行 | 占用 CPU，正在执行     |
| 阻塞 | 等待 I/O 或事件        |
| 终止 | 执行结束，资源回收     |

**状态转换**：

- 就绪 -> 运行：被调度
- 运行 -> 就绪：时间片用完
- 运行 -> 阻塞：I/O 请求
- 阻塞 -> 就绪：I/O 完成

## 可选补充

### 1. PCB（进程控制块）

> 操作系统管理进程的数据结构，存状态、程序计数器、寄存器等

**包含信息**：

- 进程标识符（PID）
- 状态（就绪、运行、阻塞）
- 程序计数器
- 寄存器
- 内存管理信息（页表指针）
- 打开文件列表

---

### 2. 上下文切换

> CPU 从一个进程切换到另一个进程时，保存和恢复寄存器、内存映射等，有开销

**触发条件**：

- 时间片用完（调度）
- 系统调用
- 中断
- 阻塞操作

**切换开销**：

- 直接开销：保存/恢复寄存器、页表切换
- 间接开销：CPU 缓存失效、TLB 清空

---

### 3. 线程模型

> 内核级线程、用户级线程（Java 线程是内核级）

| 模型           | 特点                                               | 示例                 |
| -------------- | -------------------------------------------------- | -------------------- |
| 1:1 （内核级） | 一个用户线程对应一个内核线程，并发性好，切换成本高 | Linux、Windows、Java |
| N:1（用户级）  | 多用户线程对应一个内核线程，切换快，但无法利用多核 | 早期协程             |
| M:N（混合）    | 多个用户线程映射到多个内核线程，灵活但复杂         | Go goroutine         |

**Java 线程**：

> 采用 1:1 模型，每个 Java 线程对应一个 OS 内核线程

---

### 4. 进程间通信（IPC）方式

| 方式     | 特点                       |
| -------- | -------------------------- |
| 管道     | 父子进程间，单向，字节流   |
| 消息队列 | 消息格式，异步             |
| 共享内存 | 最快，需同步机制（信号量） |
| 信号     | 异步通知（如 SIGKILL）     |
| Socket   | 可用于不同机器             |

---

### 5. 内存管理 - 内部碎片 VS 外部碎片

- 内部碎片：分配的内存块大于实际使用（如分页最后一块）
- 外部碎片：总空闲内存足够，但分散成小片段无法分配（如分段）

---

### 6. 物理内存 VS 虚拟内存

| 对比   | 物理内存               | 虚拟内存                        |
| ------ | ---------------------- | ------------------------------- |
| 地址   | 真实硬件地址           | 每个进程独立的抽象地址          |
| 大小   | RAM 大小               | 受地址总线限制（如 64 位 16EB） |
| 隔离性 | 无（进程间可相互影响） | 有                              |
| 缺页   | 无                     | 有                              |

---

### 7. 协程 VS 线程

| 对比项     | 线程                       | 协程                                    |
| ---------- | -------------------------- | --------------------------------------- |
| 调度方式   | OS 抢占式                  | 用户态协作式（主动让出）                |
| 切换成本   | 高（系统调用）             | 极低（只保存几个寄存器）                |
| 栈大小     | 固定（通常 1MB）           | 动态（可小至几 KB）                     |
| 数量       | 受系统限制（数千级）       | 可百万级                                |
| 并发模型   | 并行（多核）               | 并发（单核也可）                        |
| 代表性实现 | Java 线程、C++ std::thread | Go goroutine、Kotlin 协程、Python async |
